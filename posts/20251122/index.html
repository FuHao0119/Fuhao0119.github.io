大语言模型（LLM）与知识图谱（KG）深度协同研究报告

随着人工智能技术的发展，大语言模型（LLM）展现了惊人的通用语言理解与生成能力，但其面临“幻觉”、缺乏最新事实及不可解释性等核心痛点。与此同时，知识图谱（KG）作为结构化知识库，具备精确性与可解释性，但受限于构建成本高与自然语言交互能力弱。本报告探讨了将两者结合的协同模式，通过 **LLM-for-KG**（大模型构建图谱）和 **KG-for-LLM**（图谱增强大模型）两条路径，旨在构建一个兼具广度与精度的全面人工智能系统。

核心问题与互补优势

单一技术路径已遇瓶颈：LLM 缺乏事实锚点，容易产生“幻觉”；而 KG 构建昂贵且缺乏灵活的理解能力。两者结合旨在取长补短。

优势互补分析

维度 大语言模型 (LLM) 知识图谱 (KG)
核心能力 强大的泛化、推理、语言生成 结构化存储、精确事实、逻辑关联
主要劣势 幻觉问题、黑盒不可解释、知识滞后 构建困难（不完整）、缺乏语义理解
结合目标 利用 KG 消除幻觉 利用 LLM 自动化构建

方法一: LLM 辅助构建 KG

该路径旨在利用 LLM 强大的文本理解与信息抽取（IE）能力，实现从非结构化文本到结构化数据的自动化转换。LLM 在此扮演“**构建者 (Builder)**”的角色。

深度分析：风险与挑战

场景： 输入“林肯是美国历史上著名的总统。”

错误输出： <林肯, 出生地, 华盛顿特区>

后果： 事实是他出生于肯塔基州。模型“脑补”的错误关系会永久污染图谱。

来源 A： <乔布斯, 创始人, 苹果>

来源 B： <盖茨, 创建者, 微软>

难点： 计算机不认为“创始人”等于“创建者”。需要额外逻辑映射为统一的 `founded_by`。

方法一：特性雷达图

工作原理

  1. 输入： 原始非结构化文本（如新闻报道）。
  2. 处理： LLM 识别实体（Entity）及关系（Relation）。
  3. 输出： 标准化的三元组数据。
  4. 存储： 将数据存入图数据库或 JSON 文件。

代码实现逻辑

核心是利用 Prompt Engineering 强制模型输出严格的 JSON 格式，以便程序解析。


def extract_knowledge(text):
    system_prompt = """
    你是一个信息抽取专家。
    请严格输出 JSON 格式：
    [{"head": "...", "relation": "...", "tail": "..."}]
    """

    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": text}
        ]
    )
    return json.loads(response.choices[0].message.content)

方法二: KG 辅助增强 LLM

该路径即 **检索增强生成 (RAG)** 的一种形式，利用知识图谱作为“**外部大脑**”或“事实锚点”，约束 LLM 的生成内容，解决幻觉问题。

深度分析：优势与局限

提问： “《黑神话：悟空》是哪一年发售的？”

纯 LLM： 可能会根据旧新闻猜测 2023 年（幻觉）。

KG 增强： 检索到事实 <黑神话, 发售日, 2024-08-20>。

结果： 回答正确，且具备完全的可解释性。

提问： “冯骥这周三早餐吃了什么？”

KG 反馈： `NULL` (无相关三元组)。

结果： 系统无法回答。这避免了瞎编，但也暴露了能力上限受限于图谱的覆盖范围。

方法二：特性雷达图

工作原理

  1. 提问 (Query)： 用户提出自然语言问题。
  2. 检索 (Retrieve)： 系统在本地 KG 数据库中检索实体。
  3. 增强 (Augment)： 将检索到的三元组转化为自然语言事实。
  4. 生成 (Generate)： 将事实注入 Prompt，指令 LLM 基于事实回答。

代码实现逻辑

构建一个闭环问答系统，通过“防御性提示词”约束 LLM，优先使用检索到的上下文。


def chat_with_kg(query):
    # 1. 检索 (Retrieve)
    kg_data = load_kg()
    retrieved_context = retrieve_knowledge(query, kg_data)

    # 2. 增强 (Augment)
    if retrieved_context:
        system_prompt = f"""
        请利用以下【参考知识】回答：
        {retrieved_context}
        """
    else:
        system_prompt = "你是一个乐于助人的 AI 助手。"

    # 3. 生成 (Generate)
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "system", "content": system_prompt}, ...]
    )
    return response.choices[0].message.content

总结：双向赋能

LLM 与 KG 的结合并非简单的叠加，而是深度的**双向赋能**：

1. LLM → KG (赋能构建)

解决了知识图谱构建成本高、非结构化数据难以利用的难题，实现了知识的“**冷启动**”与自动化扩展。

2. KG → LLM (赋能应用)

解决了大模型事实不可信、逻辑不可查的难题，为生成式 AI 加上了“**事实的缰绳**”。

通过代码实践证明，这种协同模式在技术上是可行的，且能有效构建出具备更高可信度的垂直领域问答系统。未来的优化方向将集中在解决**幻觉清洗**和**图谱覆盖率提升**这两个核心难点上。

import json
from openai import OpenAI

# ================= 配置区域 =================
# DeepSeek API Key
API_KEY = "sk-xxxx"
BASE_URL = "https://api.deepseek.com"
KG_FILE = "kg_database.json"

333
client = OpenAI(api_key=API_KEY, base_url=BASE_URL)


def save_to_kg(new_triples):
    """读取现有图谱，追加新数据，并去重保存"""
    existing_data = []
    if os.path.exists(KG_FILE):
        with open(KG_FILE, 'r', encoding='utf-8') as f:
            try:
                existing_data = json.load(f)
            except json.JSONDecodeError:
                pass

    # 简单的去重逻辑
    current_set = {f"{t['head']}-{t['relation']}-{t['tail']}" for t in existing_data}
    added_count = 0

    for triple in new_triples:
        key = f"{triple['head']}-{triple['relation']}-{triple['tail']}"
        if key not in current_set:
            existing_data.append(triple)
            current_set.add(key)
            added_count += 1

    with open(KG_FILE, 'w', encoding='utf-8') as f:
        json.dump(existing_data, f, ensure_ascii=False, indent=4)

    return added_count


def extract_knowledge(text):
    """调用 DeepSeek 进行知识抽取"""
    print(f"正在思考并抽取知识，请稍候...")

    # Prompt，强制 LLM 输出纯 JSON
    system_prompt = """
    你是一个知识图谱构建专家。请从用户的文本中提取实体关系三元组。
    要求：
    1. 输出格式必须是严格的 JSON 列表：[{"head": "实体1", "relation": "关系", "tail": "实体2"}, ...]
    2. 不要输出任何 Markdown 标记（如 ```json），只输出纯文本 JSON。
    3. 如果文本中没有明确事实，返回空列表 []。
    4. 关系（relation）请尽量简练，如 "丈夫"、"位于"、"主要成分"。
    """

    try:
        response = client.chat.completions.create(
            model="deepseek-chat",
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": text}
            ],
            temperature=0.1  # 低温度保证输出格式稳定
        )

        content = response.choices[0].message.content.strip()
        # 清理可能存在的 markdown 符号
        content = content.replace("```json", "").replace("```", "")

        triples = json.loads(content)
        return triples
    except Exception as e:
        print(f"API 调用或解析出错: {e}")
        return []


if __name__ == "__main__":
    # 输入文本
    raw_text = """
    《黑神话：悟空》是一款由游戏科学公司开发的动作角色扮演游戏。
    游戏科学公司的总部位于中国杭州。
    这款游戏改编自中国古典名著《西游记》。
    冯骥是游戏科学的创始人。
    """

    print(f"--- 原始文本 ---\n{raw_text.strip()}\n")

    # 1. 抽取
    extracted_data = extract_knowledge(raw_text)

    if extracted_data:
        print(f"--- 抽取到的三元组 ---")
        for item in extracted_data:
            print(item)

        # 2. 存储
        count = save_to_kg(extracted_data)
        print(f"\n✅ 成功保存 {count} 条新知识到 '{KG_FILE}'")
    else:
        print("❌ 未能抽取到有效信息。")

SIMPLE_RAG.py:

import json
from openai import OpenAI

# ================= 配置区域 =================
API_KEY = "sk-xxx"
BASE_URL = "https://api.deepseek.com"
KG_FILE = "kg_database.json"
# ===========================================

client = OpenAI(api_key=API_KEY, base_url=BASE_URL)


def load_kg():
    """加载知识图谱文件"""
    try:
        with open(KG_FILE, 'r', encoding='utf-8') as f:
            return json.load(f)
    except FileNotFoundError:
        print(f"错误：找不到知识图谱文件 {KG_FILE}，请先运行第一个程序。")
        return []


def retrieve_knowledge(query, kg_data):
    """
    简单的检索逻辑：如果实体出现在问题中，就召回该三元组。
    实际生产中这里会使用向量数据库 (Vector DB) 或图查询语言 (Cypher)。
    """
    context = []
    print(f"🔍 正在图谱中检索...")

    for triple in kg_data:
        # 简单的关键词匹配：检查 头实体 或 尾实体 是否出现在问题中
        if triple['head'] in query or triple['tail'] in query:
            # 将结构化数据转化为自然语言描述，方便 LLM 理解
            fact_str = f"{triple['head']} 的 {triple['relation']} 是 {triple['tail']}"
            context.append(fact_str)
            print(f"   -> 命中知识: {fact_str}")

    return "\n".join(context)


def chat_with_kg(query):
    # 1. 加载图谱
    kg_data = load_kg()
    if not kg_data: return

    # 2. 检索相关知识
    retrieved_context = retrieve_knowledge(query, kg_data)

    # 3. 构建提示词 (Prompt Engineering)
    if retrieved_context:
        system_prompt = f"""
        你是一个基于知识图谱的问答助手。
        请利用以下【参考知识】来回答用户的问题。
        如果【参考知识】足以回答问题，请依据它回答。
        如果【参考知识】与问题无关，你可以利用你自己的知识回答，但要优先使用参考知识。

        【参考知识】：
        {retrieved_context}
        """
    else:
        print("   -> (未在图谱中找到直接相关的实体，将依靠模型自身知识回答)")
        system_prompt = "你是一个乐于助人的 AI 助手。"

    # 4. 调用 DeepSeek 回答
    print("🤖 模型正在生成答案...")
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": query}
        ]
    )

    return response.choices[0].message.content


# --- 主程序 ---
if __name__ == "__main__":
    print("--- 知识图谱增强问答系统 (输入 'exit' 退出) ---")

    while True:
        user_query = input("\n请输入问题: ")
        if user_query.lower() == 'exit':
            break

        answer = chat_with_kg(user_query)
        print(f"\n💡 DeepSeek 回答:\n{answer}\n")
        print("-" * 50)
